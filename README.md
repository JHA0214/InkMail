# InkMail

이북리더기에서 카카오메일을 확인할 수 있는 웹 앱(PWA)입니다. 메일 본문은 스크롤 대신 좌우 페이지 넘김 방식으로 읽습니다.

## 구조

- `backend/` — Express 서버. 브라우저가 직접 접속할 수 없는 IMAP(카카오메일)을 대신 연결해 REST API로 제공합니다.
- `public/` — 프론트엔드(PWA). 메일함 목록(`index.html`)과 페이지 넘김 리더(`reader.html`)로 구성됩니다.

## 준비

1. 카카오메일(mail.kakao.com) 설정 > POP3/IMAP에서 IMAP/SMTP 사용을 켜고 앱 비밀번호를 발급받습니다.
2. `backend/.env.example`을 `backend/.env`로 복사한 뒤 `KAKAO_EMAIL`, `KAKAO_APP_PASSWORD`를 채웁니다.
3. `ACCESS_KEY`에 이 앱에 접속할 때 쓸 비밀번호를 정합니다. 같은 Wi‑Fi의 다른 사람이 로그인 없이 메일함에 접근하지 못하도록 막는 최소한의 장치이니 꼭 설정하세요.

## 보안 설계

- 로그인하지 않으면 메일함 목록/본문 화면과 API 모두 접근할 수 없습니다(`ACCESS_KEY` 기반 세션 쿠키, httpOnly).
- 메일 HTML은 서버에서 정제됩니다: `<script>`/`<style>` 등 위험 태그 제거, 인라인 스타일은 화이트리스트 속성만 허용(`background:url()` 등 트래킹 벡터 차단), `<a target="_blank">`에는 `rel="noopener noreferrer"`를 강제해 reverse tabnabbing을 막습니다.
- 원격 이미지(http/https)는 자동으로 불러오지 않고, 리더 화면에서 "이미지 보기"를 눌러야 로드됩니다(트래킹 픽셀 방지). 메일에 내장된 `data:` 이미지는 네트워크 요청이 없으므로 즉시 보입니다.
- API 에러는 내부 정보(IMAP 에러 상세 등) 없이 일반화된 메시지만 응답하고, 상세 내용은 서버 로그에만 남습니다.
- 로그인 API(`/api/login`)는 15분당 10회로 요청을 제한해 비밀번호 무차별 대입을 막습니다.
- 기본 보안 헤더(CSP, `X-Frame-Options: DENY` 등)를 적용해 클릭재킹 등을 막습니다.

### ⚠️ 알려진 한계: HTTPS 미적용

이 서버는 현재 **평문 HTTP**로만 동작합니다. 즉 로그인 비밀번호(`ACCESS_KEY`)와 세션 쿠키, 메일 내용이 전부 암호화되지 않은 상태로 네트워크를 오갑니다. 신뢰할 수 없는 Wi‑Fi(카페, 공용 네트워크 등)에서 같은 네트워크의 누군가 패킷을 가로채면 로그인 정보와 메일 내용이 노출될 수 있습니다.

- 신뢰하는 가정용 공유기에서만 사용하는 걸 권장합니다.
- 공유기 자체를 절대 외부(포트포워딩, 터널링 등)에 노출하지 마세요.
- 이후 HTTPS를 적용하려면 `mkcert`로 로컬 인증서를 만들어 Node의 `https` 모듈에 연결하거나, Caddy/nginx 같은 reverse proxy를 앞단에 두는 방법이 있습니다.

## 실행

```bash
cd backend
npm install
npm start
```

브라우저(또는 이북리더기)에서 `http://<서버 주소>:3000`으로 접속합니다. 이북리더기와 서버가 같은 기기가 아니라면, 리더기 브라우저에서 서버를 실행 중인 PC의 IP로 접속하면 됩니다.
